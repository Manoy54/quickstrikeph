# Project Agent Instructions

This is a Next.js `16.2.4` App Router project with React `19.2.4`, TypeScript, Tailwind CSS v4, and npm.

## Use The Project Skills

Before making changes, read the relevant project-local skill files in `.codex/skills/`:

- `.codex/skills/nextjs-app-router/SKILL.md` for routes, layouts, data fetching, caching, metadata, assets, and Next config.
- `.codex/skills/react-components/SKILL.md` for components, hooks, state, effects, forms, and Server/Client Component boundaries.
- `.codex/skills/tailwind-css/SKILL.md` for styling, theme tokens, global CSS, responsive UI, dark mode, and class detection.
- `.codex/skills/typescript-quality/SKILL.md` for TypeScript strictness, props, route types, shared types, and TS config.
- `.codex/skills/testing-verification/SKILL.md` for lint/build/browser checks and test strategy.
- `.codex/skills/production-security/SKILL.md` for deployment, environment variables, secrets, auth, caching, SEO, accessibility, and performance.
- `.codex/skills/file-management/SKILL.md` for safe file creation, moves, renames, deletes, assets, generated files, env files, package files, and project organization.

## Source Standard

Use reliable sources only:

- Prefer installed docs that match this dependency version: `node_modules/next/dist/docs/`.
- Use official framework docs: `nextjs.org`, `react.dev`, `tailwindcss.com`, `typescriptlang.org`, and `eslint.org`.
- Do not copy patterns from old blog posts, Stack Overflow answers, or outdated Next.js versions when official docs are available.

## Verification

- Run `npm run lint` for source changes.
- Run `npm run build` for route, rendering, config, type, dependency, or production-sensitive changes.
- Browser-check visual and interactive work with the local dev server.
